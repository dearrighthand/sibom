# 시니어 데이팅 서비스 '시봄(Sibom' 상세 기획서

## 1. 전체 세부 기능 명세 (와이어프레임 제작 가능 수준)

### 1.1 회원가입 및 인증 Flow

#### 1.1.1 초기 진입 화면

**화면 구성:**

- 상단: 서비스 로고 + 캐치프레이즈 "60대 이후, 새로운 인연을 만나보세요"
- 중앙: 따뜻한 느낌의 시니어 커플 이미지 (AI 생성)
- 하단:
  - [카카오톡으로 시작하기] 버튼 (Yellow, 56px height)
  - [휴대폰 번호로 시작하기] 버튼 (Primary, 56px height)
  - 작은 텍스트: "이미 가입하셨나요? 로그인"

**기술 구현:**

- Next.js App Router 사용
- JWT 인증 구현(카카오 로그인 연동), 서버 Cookie 기반 인증
- 버튼 터치 영역 최소 48x48px 보장

#### 1.1.2 본인 인증 단계

**Step 1: 휴대폰 인증**

- 입력 필드: 휴대폰 번호 (자동 하이픈 삽입, 20pt 폰트)
- [인증번호 받기] 버튼
- 6자리 인증번호 입력 (각 숫자 개별 박스, 자동 포커스 이동)
- 타이머 표시: "3분 00초 남음" (큰 글씨, 빨간색)

**Step 2: 기본 정보 입력 (AI 가이드 동반)**

- 화면 좌측: AI 가이드 캐릭터 "인연이" (귀여운 어린아이 형상)
  - 말풍선: "안녕하세요! 저는 인연이에요. 편안하게 하나씩 알려주세요 😊"
- 입력 항목:
  - 이름 (실명, 플레이스홀더: "예: 김영철")
  - 생년월일 (Picker UI, 1940~1966년 범위)
  - 성별 (라디오 버튼, 남성/여성, 각 80px 높이)
  - 거주 지역 (시/구 단위 선택, 드롭다운 대신 버튼 방식)

**Step 3: 신원 증빙 (안전성 강화)**

- AI 가이드 말풍선: "안전한 만남을 위해 신분증을 확인해요. 개인정보는 저장되지 않아요."
- [신분증 촬영하기] 버튼 클릭 → 카메라 실행
- 가이드 오버레이: 신분증 위치 가이드라인 표시
- 자동 처리:
  - 얼굴 부분 자동 모자이크 (OpenCV.js 활용)
  - 주민등록번호 뒷자리 자동 마스킹
  - 생년월일만 추출하여 연령 검증

**기술 구현:**

- React Hook Form으로 단계별 폼 관리
- Zod로 입력 값 검증
- 서버에 신분증 이미지 암호화 저장
- 신분증 OCR: Google Cloud Vision API 또는 CLOVA OCR

#### 1.1.3 프로필 작성 (AI 대화형 인터페이스)

**화면 전환 방식: 챗봇 스타일**

**질문 1: 자기소개**

- AI: "김영철님, 자신을 소개하는 글을 편하게 적어주세요. 어떤 성격이신가요?"
- 입력창: 멀티라인 텍스트 (최대 200자)
- [AI 도움 받기] 버튼 클릭 시:
  - "저는 조용하고 차분한 성격입니다" → AI가 "저는 책 읽기와 산책을 즐기는 차분한 성격입니다. 대화를 나누며 깊이 있는 관계를 만들어가고 싶습니다" 로 확장
  - Gemini API로 자연스러운 문장 생성

**질문 2: 관심사 선택**

- AI: "평소 어떤 활동을 즐기시나요? 여러 개 선택해주세요"
- 버튼 그리드 (2열, 각 버튼 120x80px):
  - 🚶 산책/등산
  - 📖 독서
  - ✝️ 종교활동
  - 🎵 음악감상
  - 🎨 미술/전시
  - 🍳 요리
  - 👶 손주 돌보기
  - 🏋️ 운동/건강
  - 🌿 텃밭 가꾸기
  - ✈️ 여행
  - 🎭 공연관람
  - 📺 TV/영화
- 선택 시 버튼 색상 변경 (회색 → 초록색)

**질문 3: 원하는 만남의 형태**

- AI: "어떤 분을 만나고 싶으신가요?"
- 라디오 선택 (단일 선택):
  - "같이 취미를 즐길 친구"
  - "대화를 나눌 수 있는 동반자"
  - "재혼을 고려할 수 있는 인연"
  - "가볍게 식사나 차를 마실 지인"

**질문 4: 프로필 사진 등록**

- AI: "본인의 최근 사진을 올려주세요. 얼굴이 잘 보이는 사진이 좋아요!"
- [사진 선택하기] / [사진 촬영하기] 버튼
- 업로드 후 AI 자동 검증:
  - 얼굴 인식 여부 확인 (TensorFlow.js Face Detection)
  - 부적절한 이미지 필터링 (Gemini Vision API 활용)
  - 거부 사유: "얼굴이 잘 보이지 않아요", "다른 분의 사진 같아요" 등

**완료 화면:**

- "김영철님의 프로필이 완성되었어요! 곧 멋진 인연을 만나실 거예요 ✨"
- [인연 찾아보기] 버튼

**기술 구현:**

- 대화형 UI: Framer Motion으로 부드러운 전환 애니메이션
- 상태 관리: Zustand로 프로필 작성 진행도 저장
- AI 문장 생성: Gemini 1.5 Flash (저비용, 빠른 응답)

### 1.2 메인 화면 (홈)

#### 1.2.1 상단 네비게이션

**구성:**

- 좌측: 서비스 로고 (클릭 시 홈으로)
- 우측:
  - 🔔 알림 아이콘 (새 메시지 시 빨간 점 표시)
  - 👤 내 프로필 아이콘

#### 1.2.2 AI 추천 섹션 (메인 콘텐츠)

**카드 스와이프 방식 대신 리스트 방식 채택 (시니어 친화)**

**"오늘의 추천 인연" 섹션:**

- AI 캐릭터 인연이: "김영철님과 취미가 비슷한 분들이에요!"
- 추천 카드 (세로 스크롤, 한 화면에 1.5개 노출):

  **카드 구성 (각 카드 높이 400px):**

  ```
  ┌─────────────────────────────┐
  │   [프로필 사진 240x240px]    │
  │                             │
  │   이정숙, 63세              │
  │   📍 서울 강남구             │
  │                             │
  │   💬 "등산과 전시회를        │
  │       즐기는 활동적인        │
  │       사람입니다"           │
  │                             │
  │   공통 관심사:              │
  │   🚶 산책 🎨 미술           │
  │                             │
  │   AI 매칭 이유:             │
  │   "두 분 모두 아침 활동을   │
  │   선호하시고, 서울 시내에서 │
  │   만나기 편하세요"          │
  │                             │
  │  [관심 있어요] [다음에]     │
  └─────────────────────────────┘
  ```

**버튼 동작:**

- [관심 있어요] 클릭 → 상대방에게 '호감' 전송, "관심을 보냈어요! 상대방이 수락하면 대화를 시작할 수 있어요" 토스트 메시지
- [다음에] 클릭 → 다음 카드로 이동 (부드러운 페이드 애니메이션)

#### 1.2.3 탐색 필터 (선택적 기능)

**화면 하단 고정 버튼: [관심사로 찾기]**

클릭 시 바텀 시트 오픈:

- 연령대 선택: 60~65세 / 66~70세 / 71~75세 / 76세 이상
- 거주 지역: 현재 위치 기준 반경 선택 (5km / 10km / 20km / 전국)
- 관심사: 앞서 정의한 12가지 버튼 그리드
- [적용하기] 버튼 → 필터링된 결과 표시

**기술 구현:**

- 추천 알고리즘: Supabase Edge Function에서 벡터 유사도 계산
  - 관심사를 벡터화 (예: [산책=1, 독서=1, 종교=0, ...])
  - 코사인 유사도로 매칭 점수 산출
  - 거리 가중치 적용 (가까울수록 우선순위 상승)
- 카드 스와이프: react-tinder-card 라이브러리 (접근성 고려하여 버튼 방식 병행)

**기술 구현:**

### 1.4 채팅 화면 (핵심 기능)

#### 1.4.1 채팅방 리스트

**화면 구성:**

- 상단: "대화 중인 인연" 타이틀
- 리스트 아이템:
  ```
  ┌─────────────────────────────────────┐
  │ [사진]  이정숙                      │
  │ 60x60  안녕하세요, 반갑습니다.      │
  │        🟢 1 (읽지 않은 메시지)      │
  │        5분 전                       │
  └─────────────────────────────────────┘
  ```

#### 1.4.2 1:1 채팅방 (AI 도움 기능 강화)

**헤더:**

- 좌측: [← 뒤로가기]
- 중앙: 상대방 이름 + 프로필 사진 (작게)
- 우측: [︙ 더보기] → 메뉴 (신고하기, 차단하기, 대화 나가기)

**메시지 영역:**

- 상대방 메시지: 좌측 정렬, 밝은 회색 말풍선
- 내 메시지: 우측 정렬, 초록색 말풍선
- 폰트 크기: 18pt (설정에서 조정 가능)
- 시간 표시: 각 메시지 하단 작게

**입력 영역 (핵심 차별화):**

```
┌─────────────────────────────────────┐
│ [💡 AI 도움] [텍스트 입력창]  [전송]│
└─────────────────────────────────────┘
```

**[💡 AI 도움] 버튼 클릭 시 바텀 시트 오픈:**

**옵션 1: 대화 시작 도움**

- 상황: 첫 메시지를 보내야 할 때
- AI 제안:
  - "안녕하세요 이정숙님, 프로필을 보니 등산을 좋아하신다고 하셨는데 저도 북한산을 자주 가요. 어느 산을 자주 가시나요?"
  - "반갑습니다! 같은 지역에 사시는군요. 혹시 근처에 좋은 산책 코스 추천해주실 수 있나요?"
  - "안녕하세요, 프로필 사진이 정말 밝아 보이시네요. 저는 요즘 아침 운동을 시작했는데 이정숙님은 어떤 운동을 즐기시나요?"
- 사용자가 선택하면 텍스트 입력창에 자동 입력 (수정 가능)

**옵션 2: 답장 도움**

- 상황: 상대방 메시지에 어떻게 답할지 모를 때
- AI가 직전 대화 맥락 분석:
  - 상대방: "저는 주말에 주로 북한산에 가요"
  - AI 제안:
    - "저도 북한산 자주 가는데, 혹시 어느 코스를 좋아하세요?"
    - "북한산 경치가 정말 좋죠! 다음에 시간 맞으면 같이 가도 좋을 것 같아요"
    - "북한산이면 우이역 쪽인가요? 저는 주로 정릉 쪽으로 올라가요"

**옵션 3: 만남 제안 도움**

- 상황: 오프라인 만남을 제안하고 싶을 때
- AI 제안:
  - "대화 나눠보니 정말 좋은 분 같아요. 시간 되실 때 편한 곳에서 차 한 잔 하며 이야기 나눠요?"
  - "프로필에서 ○○를 좋아하신다고 하셨는데, 이번 주말에 같이 ○○ 하러 가실래요?"

**옵션 4: 감정 표현 도움**

- 상황: 고마움, 위로 등을 전하고 싶을 때
- AI 제안:
  - "오늘 대화 정말 즐거웠어요. 다음에 또 이야기 나눠요!"
  - "말씀 들으니 힘이 나네요. 감사합니다"

**기술 구현:**

- 채팅 백엔드: Supabase Realtime (WebSocket 기반)
- AI 대화 도움: Gemini 1.5 Pro
  - 프롬프트 엔지니어링: "60대 시니어의 정중하고 따뜻한 말투로 답변 생성. 반말 금지. 과도하게 친밀한 표현 지양."
  - 대화 히스토리 최근 10개 메시지 컨텍스트로 전달
- 메시지 암호화: TLS + Supabase Row Level Security

### 1.5 안전한 만남 주선 (오프라인 연계)

#### 1.5.1 만남 장소 추천 기능

**진입점: 채팅방 내 [만남 제안하기] 버튼**

**화면 구성:**

- Step 1: 두 사람의 거주지 중간 지점 자동 계산
  - Google Maps API로 중간 위치 산출
  - 반경 3km 내 시니어 친화 장소 필터링:
    - ☕ 조용한 카페 (체인점 우선: 스타벅스, 투썸플레이스)
    - 🍽️ 건강식 레스토랑
    - 🌳 공원 (벤치가 있는 산책로)
    - 🏛️ 문화센터

- Step 2: 장소 리스트 표시

  ```
  ┌─────────────────────────────────────┐
  │ [지도 미리보기]                     │
  │                                     │
  │ ☕ 스타벅스 ○○점                   │
  │ 📍 서울 강남구 ○○동 123           │
  │ 🚶 두 분 모두 도보 15분            │
  │ ⭐ 4.5 (조용하고 넓은 공간)        │
  │ [이 장소로 제안하기]                │
  │                                     │
  │ 🍽️ ○○ 한정식                      │
  │ ...                                │
  └─────────────────────────────────────┘
  ```

- Step 3: 제안 메시지 자동 작성
  - "이정숙님, 시간 되실 때 '스타벅스 강남점'에서 만나 이야기 나눠요. 두 분 집에서 비슷하게 가까워요. 편하신 날짜 알려주세요!"
  - 상대방 수락 시 캘린더에 일정 자동 추가 (Google Calendar API)

#### 1.5.2 안전 기능

**만남 전 안전 가이드 표시:**

- 팝업 형태로 체크리스트 제공:

## 2. AI 활용 차별화 포인트 (타 서비스 대비)

### 2.1 AI 온보딩 가이드 "인연이" (Conversational Onboarding)

**차별화 핵심:**
기존 데이팅 앱들은 복잡한 폼을 한 번에 제시하여 시니어 이탈률이 높음. 본 서비스는 **AI 캐릭터가 1:1 대화하듯 정보를 수집**하여 심리적 부담 감소.

**기술 구현:**

- **Gemini Flash를 활용한 동적 질문 생성:**
  - 사용자 답변에 따라 다음 질문을 자동 조정
  - 예: "등산을 좋아한다" 입력 시 → "어느 산을 자주 가세요?"라고 자연스럽게 추가 질문
  - 예: "종교활동"을 선택 시 → "교회/성당/절 중 어디에 다니세요?" 세부 옵션 제공

- **음성 입력 지원 (선택 사항):**
  - Web Speech API로 텍스트 입력 대신 음성으로 자기소개 작성 가능
  - STT → Gemini로 문장 교정 → 사용자 확인 후 저장

**구현 난이도:** ⭐⭐⭐ (중)

- Gemini API 연동은 간단하나, 대화 흐름 설계에 기획력 필요
- 1인 개발자 적합: Cursor AI로 프롬프트 템플릿 자동 생성 가능

### 2.2 컨텍스트 기반 AI 매칭 알고리즘 (Context-Aware Matching)

**차별화 핵심:**
단순 나이/지역 필터가 아닌, **생활 패턴, 가치관, 대화 스타일까지 분석**하여 '진짜 잘 맞는' 사람 추천.

**매칭 점수 산출 로직:**

1. **관심사 벡터 유사도 (40% 가중치)**
   - 12개 관심사를 벡터화: `[1, 0, 1, 0, ...]`
   - 코사인 유사도 계산
   - Supabase의 pgvector Extension 활용

2. **활동 시간대 매칭 (20%)**
   - 프로필 작성 시 "주로 활동하는 시간" 수집: 아침형/저녁형
   - 같은 시간대 선호 시 점수 상승

3. **원하는 만남 형태 일치도 (20%)**
   - "재혼 고려" vs "가벼운 친구" 불일치 시 매칭 제외
   - 같은 목적 선택 시 점수 상승

4. **대화 스타일 유사도 (10%)**
   - 채팅 메시지 분석 (최소 10개 이상 메시지 교환 후 적용):
     - Gemini API로 "정중한 말투", "활발한 대화", "간결한 답변" 등 스타일 분류
     - 비슷한 스타일끼리 추가 점수

5. **거리 가중치 (10%)**
   - 5km 이내: +10점

- 10km 이내: +5점
  - 20km 이내: +2점

**AI 매칭 이유 설명 (Explainable AI):**

- 추천 카드마다 "왜 이 사람을 추천했는지" 자연어로 설명:
  - "두 분 모두 아침 산책을 즐기고, 북한산 근처에 살고 계세요"
  - "이정숙님은 전시회를 좋아하는데, 김영철님도 미술에 관심이 많으시네요"
- Gemini API로 매칭 점수 데이터를 자연어 문장으로 변환

**구현 난이도:** ⭐⭐⭐⭐ (중상)

- pgvector 학습 곡선 있으나, 온라인 튜토리얼 풍부
- Cursor AI로 SQL 쿼리 자동 생성 가능

### 2.3 실시간 AI 대화 코칭 (Real-time Conversation Coach)

**차별화 핵심:**
시니어들이 가장 어려워하는 "무슨 말을 할지 모르는 상황"을 AI가 실시간 해결.

**기능 세부:**

**1) 첫 메시지 자동 생성 (Cold Start 해결)**

- 상대방 프로필 분석 → 맞춤형 첫 인사말 3가지 제안
- 예시 프롬프트:

  ```
  사용자: 김영철, 68세, 관심사: 산책, 독서
  상대방: 이정숙, 63세, 관심사: 등산, 전시회
  자기소개: "활동적인 사람입니다. 주말마다 북한산 가요"

  → Gemini 출력:
  1. "안녕하세요 이정숙님, 저도 산 좋아하는데 북한산은 어느 코스가 좋으세요?"
  2. "반갑습니다! 프로필 보니 전시회도 자주 가신다고 하셨는데 최근 어떤 전시 보셨나요?"
  3. "안녕하세요, 주말에 북한산 가신다니 부러워요. 저는 주로 동네 산책하는데 추천해주실 곳 있나요?"
  ```

**2) 대화 맥락 분석 후 답장 제안**

- 직전 대화 3~5개 턴을 Gemini에 전달
- "상대방이 질문을 했는가?" / "공감이 필요한가?" / "주제 전환이 필요한가?" 자동 판단
- 상황별 템플릿:
  - 질문 답변형: "저는 ○○에요. ○○님은 어떠세요?"
  - 공감형: "맞아요, 정말 그렇죠. 저도 비슷한 경험이 있어요"
  - 주제 전환형: "그런데 혹시 ○○도 좋아하세요?"

**3) 부적절한 메시지 경고 (Safety Filter)**

- 전송 전 AI가 메시지 검사:
  - 금전 요구 의심 문구: "돈", "송금", "빌려줘" 등 → "혹시 금전 거래를 요청하시는 건가요? 안전을 위해 권장하지 않아요" 경고
  - 과도하게 친밀한 표현: "자기야", "여보" 등 → "아직 친해지는 단계인데 조금 더 정중한 표현은 어떨까요?" 제안
- Gemini Moderation API 활용

**4) 만남 제안 타이밍 추천**

- 대화 횟수, 메시지 길이, 감정 톤 분석
- 충분히 친해졌다고 판단 시 (예: 20회 이상 메시지 교환, 긍정 키워드 빈도 높음):
  - AI 알림: "💡 이정숙님과 대화가 잘 이어지는 것 같아요. 직접 만나보는 건 어떠세요?"
  - [만남 제안하기] 버튼 활성화

**구현 난이도:** ⭐⭐⭐⭐ (중상)

- Gemini API 반복 호출 시 비용 고려 필요 → Flash 모델로 비용 최적화
- 프롬프트 엔지니어링이 핵심 (시니어 말투, 정중함 유지)

### 2.4 AI 기반 사기 탐지 시스템 (Fraud Detection)

**차별화 핵심:**
시니어 대상 로맨스 스캠 방지. **프로필과 대화 내용을 AI가 실시간 분석**하여 의심 계정 사전 차단.

**탐지 로직:**

**1) 프로필 이미지 검증**

- Google Reverse Image Search API로 이미지 출처 확인
- 웹에서 10회 이상 검색되는 이미지 → "이 사진은 인터넷에서 자주 보이는 이미지예요. 본인 사진이 맞나요?" 경고
- 얼굴 연령 추정 (TensorFlow.js Age Detection):
  - 프로필 나이 68세인데 사진은 30대로 보임 → 관리자 검토 대기열 이동

**2) 대화 패턴 분석**

- 의심 키워드 자동 탐지:
  - "투자", "수익", "가상화폐", "해외", "긴급", "병원비", "수술" 등
  - 탐지 시 사용자에게 경고 팝업: "⚠️ 금전 요구 의심 대화가 감지되었습니다. 절대 송금하지 마세요"
- Gemini로 대화 맥락 분석:
  - "짧은 시간 내 과도한 친밀감 표현" 패턴 탐지
  - 예: 3일 만에 "사랑해요", "운명이에요" 등 → 의심 점수 상승

**3) 행동 패턴 분석**

- 같은 프로필 사진으로 여러 계정 생성 시도 → IP 차단
- 가입 후 24시간 내 50명 이상에게 호감 전송 → 봇 의심

**4) 신고 누적 시스템**

- 2회 이상 신고된 계정 → 자동 프로필 비공개
- 3회 이상 → 영구 정지

**구현 난이도:** ⭐⭐⭐⭐⭐ (상)

- 다양한 API 통합 필요 (Reverse Image Search, Age Detection)
- 초기엔 키워드 기반으로 시작, 점진적으로 AI 고도화 권장

### 2.5 AI 생성 "관계 발전 리포트" (Relationship Progress Report)

**리포트 내용:**

**1) 대화 통계**

**2) 공통 관심사 발견**

- Gemini가 대화 내용 분석:
  - "두 분 모두 클래식 음악을 좋아하시네요! 다음엔 공연 함께 가보는 건 어떨까요?"

**3) 대화 스타일 분석**

- 감정 톤 분석 (Gemini Sentiment Analysis):
  - "이번 주 대화는 따뜻하고 긍정적이었어요 😊"
  - 그래프로 시각화: 긍정 85% / 중립 10% / 부정 5%

**4) 다음 단계 제안**

- "충분히 대화를 나눴으니, 이번 주말에 만나보는 건 어떨까요?"
- [만남 제안하기] 버튼 하이라이트

### 2.6 음성 메시지 + 자동 텍스트 변환 (Voice Message with Transcription)

**차별화 핵심:**
타이핑이 불편한 시니어를 위해 **음성 메시지 전송 + AI 자동 텍스트 변환** 제공. 받는 사람은 텍스트로도 확인 가능.

**기능 흐름:**

1. 채팅창에서 [🎤 음성 녹음] 버튼 클릭
2. 녹음 (최대 1분)
3. 전송 시 자동으로:
   - 음성 파일 저장 (Supabase Storage)
   - Gemini Speech-to-Text로 텍스트 변환
   - 메시지 형태:

     ```
     🎤 음성 메시지 (25초)
     [재생 버튼]

     📝 텍스트 변환:
     "안녕하세요, 오늘 날씨가 정말 좋네요.
     혹시 시간 되시면 산책하러 나가실래요?"
     ```

**추가 기능: AI 발음 교정 (선택 사항)**

- 녹음 후 "발음이 명확하지 않을 수 있어요. 다시 녹음하시겠어요?" 제안
- 노인성 난청 고려하여 상대방에게 큰 목소리로 재생

**구현 난이도:** ⭐⭐⭐ (중)

- Gemini STT API 사용

### 2.7 AI 생성 "오늘의 대화 주제" (Daily Conversation Starter)

대화가 끊겼을 때 **AI가 매일 새로운 대화 주제 자동 제안**. 사용자 관심사 기반 맞춤형 질문.

**예시:**

- 등산 좋아하는 사람: "요즘 단풍이 예쁜데, ○○님은 어느 산 추천하세요?"
- 요리 좋아하는 사람: "오늘 저녁 메뉴 뭐 드셨어요? 저는 김치찌개 끓여 먹었어요"
- 손주 돌보는 사람: "손주분들이 요즘 뭘 좋아하나요? 제 손주는 공룡 장난감에 푹 빠져 있어요"

**적용 시점:**

- 대화가 24시간 이상 없을 때 푸시 알림:
  - "💬 이정숙님에게 오늘의 대화 주제를 보내보세요!"

**구현 난이도:** ⭐⭐ (하)

- Gemini로 매일 새로운 주제 생성 (Cron Job)
- 관심사별 템플릿 DB 구축

## 3. 기술 스택 및 개발 난이도 요약

### 3.1 권장 기술 스택 (1인 개발자 최적화)

**Frontend:**

- Next.js 14 (App Router) + TypeScript
- Tailwind CSS (시니어 친화 디자인 시스템)
- Shadcn/ui (접근성 높은 컴포넌트)
- Framer Motion (부드러운 애니메이션)

**Backend:**

- Supabase (Auth, Realtime DB, Storage, Edge Functions)
- pgvector Extension (벡터 기반 매칭)

**AI:**

- Gemini 1.5 Flash (저비용, 빠른 응답)
- Gemini 1.5 Pro (복잡한 분석)
- TensorFlow.js (브라우저 내 얼굴 인식)

**결제:**

- 토스페이먼츠 API

**배포:**

- Vercel (프론트엔드)
- Supabase Edge Functions (백엔드 로직)

**개발 도구:**

- Cursor AI (코드 자동 생성)
- v0.dev (UI 프로토타이핑)

### 3.2 개발 우선순위 (MVP 기준)

**Phase 1 (필수):**

1. 회원가입/인증 (2주)
2. 프로필 작성 (AI 가이드) (1주)
3. 기본 매칭 알고리즘 (2주)
4. 1:1 채팅 (AI 대화 도움 포함) (3주)

**Phase 2 (핵심 차별화):** 5. 안전한 만남 주선 (1주) 6. 사기 탐지 시스템 (2주) 7. 프리미엄 구독 (1주)

**Phase 3 (부가 기능):** 8. 관계 발전 리포트 (1주) 9. 음성 메시지 (1주) 10. 오늘의 대화 주제 (3일)

**총 개발 기간:** 약 3~4개월 (1인 풀타임 기준)
